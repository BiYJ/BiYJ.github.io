SVN 本地客户端

Oct 31, 2017效率 | 工具点击

一、终端命令

  1. /**
    • @brief 从本地导入代码到服务器(初始化导入)。将 ~/…/MyCode 中的所有内容上传到 svn 服务器 iOS 仓库的Code 目录下
    • @param localhost 使用相对路径
    • @param -m 表示注释代码作用
  5. */
  6. $ svn import ~/…/MyCode svn://localhost/iOS/Code –username=dbb –password=123 -m “初始化导入”
  7. /**
    • @brief 从服务器端下载代码到客户端本地。将 svn 服务器iOS 仓库的内容下载到~/…/LocalCode 目录中
  9. */
  10. $ svn checkout svn://localhost/iOS –username=dbb –password=123 ~/…/LocalCode
  11. /**
    • @brief 提交更改过的代码到服务器。将 ~/…/LocalCode 下的所有修改都同步到服务器端。
    • 假如此次只修改了 main.m 文件，可以看到终端的打印信息：Sending Code/Code/main.mTransmitting file data.Committed revision
  14. */
  15. $ cd ~/…/LocalCode // 定位到 ~/…/LocalCode 目录
  16. $ svn commit -m “修改了 main.m 文件” // 输入提交命令
  17. /**
    • @brief 更新服务器端的代码到客户端
  19. */
  20. $ cd ~/…/LocalCode
  21. $ svn update
  22. $ svn help // 查看其他用法

例：SmartSVN。在客户端的地址栏中输入 svn://192.168.0.107/SVN/iOS

三、svn 服务器地址变换

  1. $ cd 项目位置 // 进入项目所在的文件夹

  2. $ svn info // 查看 svn 信息

  3. // 输出结果

  4. Path: .
  5. Working Copy Root Path: /Users/jpm/Documents/workspaces/ios/MyProject/BSChartNet
  6. URL: https://XXXX@10.25.69.13/svn/XXXX
  7. Repository Root: https://XXXX@10.25.69.13/svn/XXX
  8. Repository UUID: 485b2045-778f-e54e-8a78-595323a4869c
  9. Revision: 5562
  10. Node Kind: directory
  11. Schedule: normal
  12. Last Changed Author: dbb
  13. Last Changed Rev: 4837

  14. Last Changed Date: 2013-09-17 11:27:25 +0800 (二, 17 9 2013)

  15. // 复制红色部分，执行替换命令

  16. svn switch – relocate https://XXXX@10.25.69.13/svn/XXX https://XXXX@10.25.69.14/svn/XXX

第一次修改需要数据 svn 密码和机器密码，输入就 ok 了。
